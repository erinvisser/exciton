MARLEY KD OMP Interface (v121_kd branch)

Purpose: The v121_kd branch of the MARLEY private fork adds diagnostic getter functions, a public OMP evaluation method, and a corrected spin-orbit equation to the KoningDelarocheOpticalModel class.

These changes enable the exciton project to extract OMP parameters (well depths, geometry, KD03 coefficients, and the full radial potential) for comparison against TALYS output. The branch builds on MARLEY v1.2.1 and is located at /workspace/marley-private.

Public Methods

setIncidentEnergyAndFragment()

Original method from 4f4d65f. Computes central OMP parameters only (spin-0, l=0, j=0):

void setIncidentEnergyAndFragment(double E_kin, int fragment_pdg);

updateWellDepths()

Added in cbfb289, extended with l parameter in ea6277d. Computes all five potential depths including spin-orbit (Vso, Wso) using j = l + 1/2 (aligned spin): 

void updateWellDepths(double E_kin, int fragment_pdg, int l = 0);

evalOMP()

Added in c84ae23. Public wrapper around the private omp(double r) method, which computes U(r) = V(r) + iW(r): 

std::complex<double> evalOMP(double r) const;

Spin-Orbit Formalism

The spin-orbit eigenvalue is computed in calculate_om_parameters() as: 

// Eigenvalue of l·σ = ½[j(j+1) - l(l+1) - s(s+1)]
// With ħ = 1, the term entering the potential is:
spin_orbit_eigenvalue = 0.25 * ((two_j - two_s) * (two_j + two_s + 2))
                        - l * (l + 1);

For updateWellDepths() with l = 0: j = 1/2, so l·σ = 0 and the spin-orbit term vanishes. The central potential geometry terms (Rv, av, Rd, ad) are unaffected by the spin choice.

Commit 372cddd corrected a factor-of-2 error in the spin eigenvalue equation. The validated formula produces spin-orbit eigenvalues that match TALYS.

Usage in Exciton Code

// Create KD model for target nucleus
marley::KoningDelarocheOpticalModel kd(target_Z, target_A);

// For central potential only (preeqmode=3):
kd.setIncidentEnergyAndFragment(kinetic_energy, pdg_code);

// For full potential including spin-orbit:
kd.updateWellDepths(kinetic_energy, pdg_code, /* l = */ 0);

// Extract parameters
double Wv = kd.getWv();
double Wd = kd.getWd();
double wvol = wvolRadialIntegral(kd);

// Evaluate radial potential at r = 5.0 fm
std::complex<double> U = kd.evalOMP(5.0);
