Output Format

Both executables produce output in the YANDF-0.4 (Yet Another Nuclear Data Format) metadata header followed by formatted data tables.

Metadata Header

Lines starting with # contain metadata. Example from lambda:

# title: Ar40(n,x) two-component exciton model
# source: exciton (C++), numerical mode, clenshaw-curtis, guarded integration, matrix-element kernel
# user: Erin Visser
# date: 2026-06-16
# format: YANDF-0.4
# target:
# Z: 18
# A: 40
# nuclide: Ar40
# parameters:
# E-incident [MeV]: 2.000000E+01
# E-compound [MeV]: 25.60156
# Fermi well depth [MeV]: 38.0
# ...
# columns: 8
# entries: 84

Lambda (Transition Rate) — 8-Column Format

Two header lines (with ##) precede the data:

## p(p) h(p) p(n) h(n) lambdapiplus lambdanuplus lambdapinu lambdanupi
## [] [] [] [] [sec^-1] [sec^-1] [sec^-1] [sec^-1]
ColNameWidthUnitsDescription
1p(p)14—Proton particle number pπ
2h(p)14—Proton hole number hπ
3p(n)14—Neutron particle number pν
4h(n)14—Neutron hole number hν
5lambdapiplus15s-1Proton pair creation rate \(\lambda_{\pi^+}\)
6lambdanuplus15s-1Neutron pair creation rate \(\lambda_{\nu^+}\)
7lambdapinu15s-1Proton→neutron conversion \(\lambda_{\pi\nu}\)
8lambdanupi15s-1Neutron→proton conversion \(\lambda_{\nu\pi}\)

Omega (State Density) — 13-Column Format

## p(p) h(p) p(n) h(n) n E_x U pairing P_ph n_crit g_p g_n density
## [] [] [] [] [] [MeV] [MeV] [MeV] [] [] [MeV^-1] [MeV^-1] [MeV^-1]
ColNameWidthDescription
1–4pπ, hπ, pν, hν14Exciton configuration
5n14Total exciton number
6Ex15Excitation energy [MeV]
7U15Available energy after pairing [MeV]
8pairing12Pairing energy Δ [MeV]
9Pph12Pairing correction factor
10ncrit12Critical exciton number
11gp14Proton single-particle density [MeV-1]
12gn14Neutron single-particle density [MeV-1]
13density15ω(p,h,E) [MeV-1]

Numerical Format

All floating-point values are printed using:

std::cout << std::scientific << std::uppercase;
std::cout.precision(6);

Format: ±X.XXXXXXE±XX (scientific notation, 6 decimal places, uppercase E).

OMP Kernel Header (OMP branch)

When using --kernel optical-model, additional metadata lines appear:

# OMP constant C_omp: 0.55
# M2 constant (C1): 1.000000E+00
# Rpinu: 1.000000E+00
# Wompfac(same-type): 1.833333E-01
# Wompfac(cross-type): 3.666667E-01
# Wompfac(conversion): 2.750000E-01

Comparison Scripts

The Python scripts expect the same YANDF-0.4 format from both exciton and TALYS:

  • compare_omega.py — parses 13-column format, computes relative error per entry
  • compare_lambda.py — parses 8-column format, computes relative error per entry
