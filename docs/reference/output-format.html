YANDF-0.4 Output Format

Both executables produce output in the YANDF-0.4 (Yet Another Nuclear Data Format) text format.

Format Specification

YANDF-0.4 is a flat text format with three element types:

ElementSyntaxPurpose
Comment# textMetadata headers, column descriptions
Header## column headersColumn names and units on consecutive lines
Datawhitespace-separated numbersNumerical data in scientific notation

Lambda Output File Naming

Lambda output files follow the pattern:

lambda_Ar40_n_<mode>_<method><guard>_<kernel>_<timestamp>.log

Where:

  • mode = analytical or numerical
  • method = clenshaw-curtis or midpoint
  • guard = _guarded or empty
  • kernel = matrix-element or optical-model
  • timestamp = YYYYMMDD_HHMMSS

Output Directory Structure

Lambda output files are organized into subdirectories under lambda_printouts/OMP/:

lambda_printouts/OMP/
KD03_Output/ kd03_Ar40_n_*.log — KD03 coefficient dump
Well_Depths_Output/ well_depths_Ar40_n_*.log
Radial_Potential_Output/ radpot_Ar40_n_*.log
Rates_Output/ lambda_Ar40_n_*.log — Transition rates

Parse Script Compatibility

The Python comparison scripts parse YANDF-0.4 format by:

  1. Skipping lines beginning with # or ##
  2. Splitting remaining lines into whitespace-separated tokens
  3. Converting tokens to float
  4. Grouping rows by the preceding # quantity: type: ... header

This means blank lines, extra comments, and trailing whitespace are tolerated.
