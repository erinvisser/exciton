Validation Modes

The lambda executable has three built-in validation modes to verify the Koning-Delaroche OMP implementation against TALYS and MARLEY.

1. KD03 Parameter Validation (--mode kd03)

This is the default mode. It performs three diagnostics:

1a. KD03 Coefficient Dump

Dumps all Koning-Delaroche optical model coefficients from the MARLEY KoningDelarocheOpticalModel constructor for Ar-40 (Z=18, A=40):

$ make run ARGS="--mode kd03"

Output file: lambda_printouts/OMP/KD03_Output/kd03_Ar40_n_*.log

Contents include:

  • Neutron and proton coefficients: v1–v4, w1–w2, d1–d3, vso1–vso2, wso1–wso2
  • Fermi energies Efn and Efp
  • Geometry parameters: Rv, av, Rd, ad, Rso, aso in both absolute (fm) and reduced (fm) forms
  • Coulomb parameters (proton): Vcbar, Rc

1b. MARLEY Q-Values

Q-values for all open channels on Ar-40 + n, computed from the MARLEY MassTable:

\[ Q_{(n,\gamma)} = M_{\text{Ar-40}} + M_n - M_{\text{Ar-41}} \]

The MassTable uses experimental atomic masses from the Atomic Mass Evaluation (AME), with liquid-drop model fallback by Koning et al. (2008). Q-values are computed for:

ChannelReactionFormula
(n,γ)n + Ar-40 → Ar-41 + γMEAr40 + MEn − MEAr41
(n,p)n + Ar-40 → p + K-40MEAr40 + MEn − MEH1 − MEK40
(n,d)n + Ar-40 → d + Cl-39MEAr40 + MEn − MEd − MECl39
(n,t)n + Ar-40 → t + Cl-38MEAr40 + MEn − MEt − MECl38
(n,3He)n + Ar-40 → 3He + S-38MEAr40 + MEn − MEh − MES38
(n,α)n + Ar-40 → α + S-37MEAr40 + MEn − MEα − MES37

The atomic mass unit u = 931.494061 MeV is used for converting mass excesses.

1c. Energy-Dependent OMP Table (TALYS flagomponly format)

For each projectile type (neutron and proton), prints the KD03 parameters at selected energies from −8.0 to 20.0 MeV in TALYS-compatible format.

2. Well Depths Validation (--mode well_depths)

Computes the six well depths as functions of energy:

$ make run ARGS="--mode well_depths"

Output file: lambda_printouts/OMP/Well_Depths_Output/well_depths_Ar40_n_*.log

Columns: E(MeV) k Vv(MeV) Wv(MeV) Wd(MeV) Vso(MeV) Wso(MeV) Rv(fm) av(fm) Rd(fm) ad(fm) Rso(fm) aso(fm)

where k = 1 (neutron) or 2 (proton). Validated against TALYS flagomponly y output.

3. Radial Potential Validation (--mode radial_potential)

Computes the complex radial profile of the optical potential U(r):

$ make run ARGS="--mode radial_potential --l 0"
$ make run ARGS="--mode radial_potential --l 2" # with d-wave SO contribution

Output file: lambda_printouts/OMP/Radial_Potential_Output/radpot_Ar40_n_*.log

Columns: E(MeV) k r(fm) Re(U)(MeV) Im(U)(MeV)

The potential includes the spin-orbit contribution: \(U(r) = V(r) + V_{so}(r)\langle\mathbf{l}\cdot\mathbf{s}\rangle + i[W(r) + W_{so}(r)\langle\mathbf{l}\cdot\mathbf{s}\rangle]\).

Validation against TALYS: The Python script talys_total_omp/talys_omp_total.py runs TALYS and extracts the same radial profiles for comparison:
$ python3 talys_total_omp/talys_omp_total.py --energies 20.0 --ls 0 -o talys_u.dat

4. Comparison Scripts

Lambda Comparison

$ python3 compare_lambda.py \
lambda_printouts/OMP/Rates_Output/lambda_ref.txt \
lambda_printouts/OMP/Rates_Output/lambda_new.txt \
comparison_report.txt 1e-4

Omega Comparison

$ python3 compare_omega.py \
omega_printouts/omega_ref.txt \
omega_printouts/omega_new.txt \
omega_comparison.txt 1e-4

TALYS OMP Extraction

For cross-checking radial potential profiles:

$ python3 talys_total_omp/talys_omp_total.py \
--energies 0.0,5.0,10.0,15.0,20.0 \
--element ar --mass 40 --projectile n \
--ls 0 -o talys_omp_comparison.dat
