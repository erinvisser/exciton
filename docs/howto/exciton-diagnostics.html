Exciton Diagnostics

Purpose: Print OMP parameters, Pauli integration bounds, and transition-rate intermediate values from the exciton C++ code for comparison against TALYS diagnostic output.

OUR_OMP Diagnostic Format

The exciton code prints OMP parameter diagnostics in a format matching the TALYS TALYS_OMP lines, prefixed with OUR_OMP: 

# OUR_OMP Z=20 A=40 e=-8 pdg=2212 Wv=5.27935e-05 Wd=0.00250571
  Rv=4.05388 av=0.671852 Rd=4.40556 ad=0.53952 wvol=0.00121917
# OUR_OMP Z=20 A=40 e=-8 pdg=2112 Wv=0.0107876 Wd=0.549063
  Rv=4.05388 av=0.671852 Rd=4.40556 ad=0.537976 wvol=0.266143

Fields:

Field Description
Z Target atomic number
A Target mass number
e Kinetic energy (MeV)
pdg Particle PDG code (2212=proton, 2112=neutron)
Wv Imaginary volume well depth (MeV)
Wd Imaginary surface well depth (MeV)
Rv Volume radius parameter (fm)
av Volume diffuseness (fm)
Rd Surface radius parameter (fm)
ad Surface diffuseness (fm)
wvol Volume-averaged absorptive potential ⟨W⟩ (MeV)

These lines are generated by calling KoningDelarocheOpticalModel::setIncidentEnergyAndFragment() (for central potential only, spin-0) or updateWellDepths() (for full potential including spin-orbit, spin-1/2) at each energy point, then computing ⟨W⟩ via wvolRadialIntegral(). The output is written to lambda_printouts/previous/omp_diag.txt.

L1 / L2 Pauli Integration Bounds

The TALYS lambda subroutines (lambdapiplus.f90, lambdanuplus.f90) compute Pauli-prescribed integration bounds L1 and L2 for each transition channel. These define the range of intermediate excitation energy u over which the integrand is evaluated: 

L1 = APauli(pπ, hπ, pν+1, hν+1) -
     APauli(pπ, hπ, pν-1, hν)
L2 = U - APauli(pπ, hπ, pν-1, hν)

where APauli is the available excitation energy after Pauli correction. The exciton code implements L1/L2 equivalently (see transition_rates.cpp): 

// For neutron pair creation (ν⁺):
L1nup = Apauli2(ppi, hpi, pnu + 1, hnu + 1) - Apauli2(ppi, hpi, pnu - 1, hnu);
L2nup = U - Apauli2(ppi, hpi, pnu - 1, hnu);

Four pairs of bounds are computed per exciton state:

Bounds Channel TALYS subroutine
L1ν⁺, L2ν⁺ Neutron pair creation lambdanuplus.f90
L1νh, L2νh Neutron hole creation lambdanuplus.f90
L1π⁺, L2π⁺ Proton pair creation lambdapiplus.f90
L1πh, L2πh Proton hole creation lambdapiplus.f90

Transition Rate Diagnostics (wvol, Weff, lamcol)

The exciton model prints intermediate OMP values for each transition rate in a format matching the TALYS TALYS_DIAG lines: 

state=(pπ, hπ, pν, hν)
  k=1 S=... uu=... eopt=... nen=... wvol=... Weff=... lamcol=... [ratio=...]

Key intermediate quantities:

Quantity Formula Description
S MassTable::get_fragment_separation_energy() Separation energy for emitted nucleon
eopt max(uu - S, -20) Kinetic energy used for OMP lookup
wvol ⟨W⟩(eopt) Volume-averaged absorptive potential
Weff Wompfac × wvol Effective potential weighted by OMP constant
lamcol 2 · Weff / ħ Collision rate (s-1)
ratio ωhole / ωparticle Particle-hole density ratio (conversion only)

Comparison Scripts

Two Python comparison scripts live at the project root:

Script Purpose
compare_lambda.py Compare two YANDF-0.4 lambda files (8-column transition rate data blocks). Outputs absolute difference and relative error (%) tables. Usage: python3 compare_lambda.py <file1> <file2> <output> [tolerance]
compare_omega.py Compare two ph_density.out files. Handles both particle-hole state density and spin distribution blocks. Outputs absolute difference and relative error tables. Usage: python3 compare_omega.py <file1> <file2> <output> [tolerance]

Workflow

To generate diagnostic output from the exciton code:

# Build the code
make

# Run lambda calculation with OMP kernel
./build/lambda --kernel optical-model

# Compare against TALYS reference
python3 compare_lambda.py \
  lambda_printouts/previous/lambda_neutron_Ar40_numerical.txt \
  /path/to/talys/lambda.out \
  comparison_report.txt
