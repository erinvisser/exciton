Lambda — Internal Transition Rate Calculations

How to compute, interpret, and compare internal transition rates for the two-component exciton model.

Basic Usage

$ make run # default: matrix-element kernel, kd03 mode

Or directly:

$ MARLEY=../marley-private ./executables/lambda

Matrix-Element Kernel (Kalbach-Cline M²)

The default collision kernel uses the constant matrix element approximation (Eq. 13.24–13.26):

\[ M^2 = C_1\,\frac{A_p}{A^3}\, \left[7.48\,C_2 + \frac{4.62\times10^5}{\bigl(10.7\,C_3 + \frac{E_{\text{tot}}}{n A_p}\bigr)^3}\right] \]

Channel ratios scale M² for each channel type:

$ make run ARGS="--kernel matrix-element"

The output header shows the channel ratios used:

# Constant for matrix element: 1.000000E+00
# p-p ratio for matrix element: 1.000000E+00
# n-n ratio for matrix element: 1.500000E+00
# p-n ratio for matrix element: 1.000000E+00
# n-p ratio for matrix element: 1.000000E+00

Analytical Mode

The analytical formulas (Eq. 13.27) provide closed-form transition rates:

$ make run ARGS="--analytical"
Note: An analytical factor of 1.20 is applied to M² in analytical mode. For n=1 states in numerical mode, the 1.20 factor is cancelled to match TALYS preeqmode=1.

Optical Model Kernel (TALYS preeqmode=3)

The OMP kernel replaces constant M² with a collision probability from the Koning-Delaroche optical model:

$ make run ARGS="--kernel optical-model"

This uses three wompfac values to bridge M² and OMP normalizations:

\[ \begin{aligned} \text{wompfac}_{\text{same}} &= \frac{M^2_c \cdot 0.55}{1 + 2R_{\pi\nu}/R_{\pi\pi}} \\[4pt] \text{wompfac}_{\text{cross}} &= \frac{M^2_c \cdot 0.55 \cdot 2R_{\pi\nu}/R_{\pi\pi}}{1 + 2R_{\pi\nu}/R_{\pi\pi}} \\[4pt] \text{wompfac}_{\text{conv}} &= \tfrac12(\text{wompfac}_{\text{same}} + \text{wompfac}_{\text{cross}}) \end{aligned} \]

The OMP kernel automatically uses numerical integration (Clenshaw-Curtis via MARLEY's Integrator, or midpoint).

Integration Method Options

Clenshaw-Curtis (default)

Uses MARLEY's adaptive Clenshaw-Curtis quadrature with 50-point base grid:

$ make run ARGS="--kernel optical-model --method clenshaw-curtis"
Guard bounds are applied by default to prevent the integration from sampling unphysical regions where the energy is below the Pauli blocking threshold. Disable with --no-guards.

Midpoint Rule

Simple midpoint Riemann sum with configurable bin count:

$ make run ARGS="--method midpoint --midbins 200"

Understanding the Output

The lambda executable outputs transition rates for 21 exciton states. For each state, four rates are computed:

RateProcessEquation
\(\lambda_{\pi^+}\)Proton pair creation: \(\pi + \nu \rightarrow \pi + \pi + \pi^{-1} + \nu\)Eq. 13.17 (numerical) / Eq. 13.27 (analytical)
\(\lambda_{\nu^+}\)Neutron pair creation: \(\pi + \nu \rightarrow \pi + \nu + \nu + \nu^{-1}\)Eq. 13.18 (numerical) / Eq. 13.27 (analytical)
\(\lambda_{\pi\nu}\)Proton-to-neutron conversion: \(\pi + \nu \rightarrow \nu + \nu + \pi^{-1}\)Eq. 13.19 (numerical) / Eq. 13.27 (analytical)
\(\lambda_{\nu\pi}\)Neutron-to-proton conversion: \(\pi + \nu \rightarrow \pi + \pi + \nu^{-1}\)Eq. 13.19 (numerical) / Eq. 13.27 (analytical)

Comparing Lambda Outputs

$ python3 compare_lambda.py reference.txt new.txt report.txt
$ python3 compare_lambda.py ref.txt new.txt report.txt 1e-6

Typical Results for n + Ar-40

With default parameters (Einc = 20.0 MeV, Ecomp = 25.60156 MeV, V = 38.0 MeV):

  • Pair creation rates (\(\lambda^+\)) dominate for low exciton numbers (n ≤ 4)
  • Conversion rates (\(\lambda^{\pi\nu}\), \(\lambda^{\nu\pi}\)) become competitive at higher n
  • OMP kernel generally produces lower rates than M² kernel for n+Ar-40 at 20 MeV
