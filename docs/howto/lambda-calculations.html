Lambda (Transition Rate) Calculations

The lambda calculation evaluates the four internal transition and conversion rates for each exciton state. The implementation is in src/transition_rates.cpp and the main executable is src/main_lambda.cpp.

The Four Rates

Symbol Name Type Description
\(\lambda_{\pi^+}\) lambdapiplus Pair creation Proton emission creates a new proton–hole pair
\(\lambda_{\nu^+}\) lambdanuplus Pair creation Neutron emission creates a new neutron–hole pair
\(\lambda_{\pi\nu}\) lambdapinu Conversion Proton converts to a neutron (p→ν)
\(\lambda_{\nu\pi}\) lambdanupi Conversion Neutron converts to a proton (ν→π)

Calculation Mode

The code supports two calculation modes, following TALYS:

  • Analytical (--analytical): Uses closed-form Eqs. 13.27 and 13.28 directly. Fast but less accurate for low n.
  • Numerical (default): Integrates Eqs. 13.17–13.19 over intermediate excitation energy e using Clenshaw–Curtis adaptive quadrature or midpoint rule.

When \(n=1\), the analytical formula is always used (the numerical integrand for n=1 has a particular form that maps to the analytical result). The 1.20 M² factor is cancelled in this case per TALYS convention (line 475 of transition_rates.cpp).

Analytical Formulas

Pair creation (Eq. 13.27):

\[ \lambda_{\pi^+}^{\text{an}}(p,h,E) = \frac{2\pi}{\hbar} \cdot \frac{g_\pi^2 (U - P_{n+1})^{n+1}}{2 n (n+1) (U-P_n)^{n-1}} \cdot \bigl[ n_\pi g_\pi M_{\pi\pi}^2 + 2 n_\nu g_\nu M_{\pi\nu}^2 \bigr] \cdot f_{\text{well}}(p+1,h+1,U,V) \]

Conversion (Eq. 13.28):

\[ \lambda_{\pi\nu}^{\text{an}}(p,h,E) = \frac{2\pi}{\hbar} \cdot M_{\pi\nu}^2 \cdot \frac{p_\pi h_\pi}{n} \cdot g_\nu^2 \cdot \frac{(U - B_{\pi\nu})^{n-1}}{(U - P_n)^{n-1}} \cdot \left[ 2 (U - B_{\pi\nu}) + n |P_n - P_{n-1}^{\pi\nu}| \right] \cdot f_{\text{well}}(p,h,U,V) \]

where \(P_n\) is the Pauli correction for the current state and \(B\) is the conversion-specific Pauli correction (maximum of before and after configurations).

Numerical Integration

The numerical approach integrates over the intermediate excitation energy e of the created or converted pair (Eq. 13.17 for pair creation):

\[ \lambda_{\pi^+}^{\text{num}}(p,h,E) = \frac{1}{\omega_i(U)} \sum_{\text{sub-channels}} \int_{L_1}^{L_2} de\, \frac{2\pi}{\hbar} M_{\text{sub}}^2\, \omega_t(e)\, g_{\text{factor}}\, \omega_r(U-e) \]

The four sub-channels for proton pair creation are:

  • pip (π particle, same-type): t=(2,1,0,0), r=(pπ-1, hπ, pν, hν), M²=M2ππ, gfactor=gπ
  • pih (π hole, same-type): t=(1,2,0,0), r=(pπ, hπ-1, pν, hν)
  • nup (ν particle, cross-type): t=(1,1,1,0), r=(pπ, hπ, pν-1, hν), M²=M2νπ, gfactor=gν
  • nuh (ν hole, cross-type): t=(1,1,0,1), r=(pπ, hπ, pν, hν-1)

Integration Methods

Method Default Description
Clenshaw–Curtis ✅ (default) Adaptive quadrature via MARLEY's num_integrate(). 50-point initial grid, automatically refined.
Midpoint — Fixed N-bin midpoint rule. Use with --method midpoint --midbins N.

Guard Bounds

When guard bounds are enabled (default), integration limits are adjusted to enforce the Pauli principle:

\[ L_1^{\text{guarded}} = \max(L_1,\; \text{guard}_{\text{created}}), \qquad L_2^{\text{guarded}} = \min(L_2,\; U - \text{guard}_{\text{removed}}) \]

where \(\text{guard} = \max(p,h)^2/g_\pi + \max(p_\nu,h_\nu)^2/g_\nu\).

Output (8-col YANDF-0.4)

Columns: pπ hπ pν hν λπ+ λν+ λπν λνπ in units of s-1.
