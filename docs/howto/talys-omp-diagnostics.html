TALYS OMP Diagnostics

Purpose: Extract OMP parameters, volume-averaged absorptive potential ⟨W⟩, and transition-rate intermediate values from TALYS to validate the exciton OMP kernel.

TALYS 2.2 source files were modified to print diagnostic output for the pre-equilibrium OMP calculation chain. These modifications are in the n-Ar40-preeq sample directory (/workspace/talys/samples/n-Ar40-preeq/). The talys.inp input sets outkd y to force global KD03 and enable parameter output.

Modified Source Files

bonetti.f90 — Volume-averaged ⟨W⟩

A write(*,'("TALYS_OMP ...")') statement was added to the phenomenological OMP loop to print OMP parameters and the volume-averaged absorptive potential at every energy point: 

k    — particle type (1=proton, 2=neutron)
nen  — energy index (0.1 MeV steps, -80 to nenend)
e    — energy (MeV)
w    — imaginary volume depth (MeV)
wd   — imaginary surface depth (MeV)
rw   — imaginary volume radius parameter (fm)
aw   — imaginary volume diffuseness (fm)
rwd  — imaginary surface radius parameter (fm)
awd  — imaginary surface diffuseness (fm)
radv — volume radius = rw · A1/3 (fm)
radd — surface radius = rwd · A1/3 (fm)
wvol — volume-averaged absorptive potential ⟨W⟩ (MeV)

The radial integration uses 50 bins with dr = 20/50 = 0.4 fm, matching TALYS bonetti.f90 line 109–110. The output is captured in talys_wvol.txt (702 lines, k=1 from nen=-80 to 270, k=2 from nen=-80 to 270).

lambdapiplus.f90 / lambdanuplus.f90 — Pair creation rates

Each transition-rate subroutine was modified to print diagnostic data for the first integration bin (i==1): 

TALYS_DIAG lambdapiplus state=ppi hpi pnu hnu j=k S=... uu=...
  eopt=... nen=... wvol=... Weff=... lamcol=... [ratio=...]

state  — exciton configuration (pπ, hπ, pν, hν)
j      — channel branch (1=1p, 2=1h for creation; 3=1p, 4=1h for conversion)
k      — particle type (1=proton, 2=neutron)
Zix    — proton index
Nix    — neutron index
S      — separation energy (MeV)
uu     — intermediate excitation energy (MeV)
eopt   — kinetic energy for OMP lookup = max(uu - S, -20) (MeV)
nen    — energy index from eopt
wvol   — volume-averaged ⟨W⟩ at eopt (MeV)
Weff   — effective potential = Wompfac × wvol (MeV)
lamcol — collision rate = 2 · Weff / ħ (s-1)
ratio  — hole/particle density ratio (conversion channels only)

These diagnostics appear in diagnostic.log (2.8 MB, ~700 TALYS_OMP lines + ~8400 TALYS_DIAG lines).

lambdapinu.f90 / lambdanupi.f90 — Conversion rates

Neutron→proton and proton→neutron conversion subroutines print the same diagnostic fields as pair creation, but with only the 1p branch (k = 2 for π→ν, k=1 for ν→π). The ratio field is always printed.

opticalnp.f90 — KD03 coefficient output

When flagoutkd is set (via outkd y), the KD03 coefficients, well depths, and geometry parameters are printed to stdout. This output is captured in the diagnostic.log header.

Output Files

Comparison with Exciton Output

The exciton code prints matching diagnostics via the omp_diag.txt output file (see Exciton Diagnostics). The TALYS_OMP and OUR_OMP lines share identical field semantics, enabling direct line-by-line comparison of:

  • Wv, Wd, Rv, av, Rd, ad — KD03 well depths and geometry
  • wvol — volume-averaged absorptive potential
  • OMP intermediate values — eopt, Weff, lamcol, ratio

The TALYS OMP diagnostic prints are enabled by a modified bonetti.f90 that prints unconditionally (no flag guard). An example invocation:

# Run TALYS with the diagnostic input
cd /workspace/talys/samples/n-Ar40-preeq
talys < talys.inp > xtra_diagnostics/diagnostic.log
