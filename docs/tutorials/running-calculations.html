Running Calculations

Omega (State Density) Calculations

The executables/omega binary computes particle–hole state densities for all (pπ, hπ, pν, hν) configurations of the two-component exciton model.

Command

$ ./executables/omega

Outputs a 13-column table in YANDF-0.4 format (see output format). The code enumerates all (pπ, hπ, pν, hν) with k=(p+h)/2 from 0 to 5 (300 states).

Flags

Flag Default Description
(none) — Runs with hard-coded parameters for Ar40(n,x)
Note: The omega executable uses fixed parameters (Z=18, N=23, V=38.0 MeV, Ecomp=25.60156 MeV) matching the standard Ar40(n,x) system. These are compiled into main_omega.cpp.

Lambda (Transition Rate) Calculations

The executables/lambda binary computes the four internal transition rates \(\lambda_{\pi^+}, \lambda_{\nu^+}, \lambda_{\pi\nu}, \lambda_{\nu\pi}\) for each exciton state.

Command

$ ./executables/lambda

Outputs an 8-column table (4 state columns + 4 rate columns) for all 84 exciton states.

Flags

Flag Default Description
--analytical numerical Use closed-form analytical expressions (Eq. 13.27) instead of numerical integration
--no-guards guarded Disable Pauli-prescribed guard bounds on integration limits
--method midpoint clenshaw-curtis Use midpoint rule with fixed bins instead of adaptive Clenshaw–Curtis quadrature
--midbins N 50 Number of midpoint bins (only with --method midpoint)

Example: Analytical Mode

$ ./executables/lambda --analytical

Example: Midpoint with 20 bins

$ ./executables/lambda --method midpoint --midbins 20

OMP Kernel (OMP branch only)

On the OMP branch, the optical model kernel can be selected at runtime:

$ ./executables/lambda --kernel optical-model

When using the OMP kernel, --analytical is overridden to numerical mode. The header output includes the wompfac values. See the OMP implementation page for details.

Validation Scripts

Two Python comparison scripts parse TALYS and exciton output:

Script Format Columns
compare_omega.py YANDF-0.4 (13-col) Compares all 300 state densities
compare_lambda.py YANDF-0.4 (8-col) Compares all 84 transition rates

See the Validations how-to for detailed usage.
