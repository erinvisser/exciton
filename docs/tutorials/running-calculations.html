Running Calculations

This tutorial covers all the ways to run the exciton code, from basic invocations to advanced options.

The Makefile Interface

The project uses a Makefile with targets:

TargetDescription
make or make allBuild both executables
make runRun lambda with MARLEY set automatically
make run ARGS="..."Run lambda with custom CLI flags
make cleanRemove both executables

Running the Omega Executable

The omega executable has no CLI arguments and hardcodes the n + Ar-40 target system:

$ ./executables/omega > omega_Ar40.txt

Output contains two data blocks:

  1. Particle-hole state densities — 25 rows (Ex = 1 to 25 MeV), 13 columns (Ex, pairing correction, gp, gn, 9 exciton configurations)
  2. Spin distribution — 12 rows (n = 1 to 12), 11 columns (n, J=0 through J=8, Sum)

Running the Lambda Executable

The lambda executable requires the MARLEY environment variable:

$ MARLEY=../marley-private ./executables/lambda [options]
# Or equivalently:
$ make run [ARGS="options"]

Example Workflows

Matrix element kernel (numerical integration)

$ make run ARGS="--kernel matrix-element --method clenshaw-curtis"

Optical model kernel

$ make run ARGS="--kernel optical-model"
Note: OMP kernel overrides --analytical — the OMP requires numerical integration.

Midpoint integration

$ make run ARGS="--method midpoint --midbins 100"

Analytical formulas only

$ make run ARGS="--analytical"

Well depth validation

$ make run ARGS="--mode well_depths"

Radial potential validation

$ make run ARGS="--mode radial_potential --l 2"
