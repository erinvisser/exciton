Getting Started

Prerequisites

  • C++17 compiler (GCC 9+, Clang 10+)
  • MARLEY library (model-dependent; see branch requirements below)
  • GSL (GNU Scientific Library)
  • ROOT (required by MARLEY build)
  • Python 3 for validation scripts (compare_omega.py, compare_lambda.py)

Building

$ cd /path/to/exciton
$ make

The Makefile compiles two executables:

Executable Source Purpose
executables/omega src/main_omega.cpp Compute particle–hole state densities \(\omega(p,h,E)\)
executables/lambda src/main_lambda.cpp Compute internal transition and conversion rates \(\lambda\)

MARLEY Dependency

The code uses MARLEY for integration and (on the OMP branch) optical model potentials. Set the MARLEY environment variable to the marley-private directory:

$ export MARLEY=/path/to/marley-private
$ make run

The make run target sets MARLEY=$(abspath ../marley-private) automatically.

Branch Overview

This project has four git branches; the documentation site covers all of them:

Branch Kernel TALYS Mode Validation
MatrixSquared Constant M² matrix element preeqmode=1 ✅ Validated (3/84 numerical outliers)
OMPfromScratch OMP kernel rebuilt from scratch preeqmode=3 ❌ OMP rates fail
OMP Koning–Delaroche optical model preeqmode=3 ⚠️ KD params pass; rates fail
main Historic (superseded) — ❌ Stale

See the full Branch Status page for details.

Running Your First Calculation

$ make
$ ./executables/lambda
# title: Ar40(n,x) two-component exciton model
# source: exciton (C++), numerical mode, clenshaw-curtis, guarded integration, matrix-element kernel
...
0 0 1 0 2.567310E+03 1.467402E+04 3.938975E+03 1.873780E+04

This produces 84 rows of transition rates in YANDF-0.4 format (8-column). See the Output Format page for column definitions.
