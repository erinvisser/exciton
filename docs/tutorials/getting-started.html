Getting Started with exciton

This tutorial walks you through your first encounter with the exciton code — from prerequisites to running your first calculation.

Prerequisites

  • C++17 compiler (g++ or clang)
  • GNU Scientific Library (GSL)
  • ROOT (CERN data analysis framework)
  • MARLEY library at ../marley-private/
  • Python 3 (for comparison scripts)

Building the Code

Navigate to the project root and run:

$ cd /workspace/exciton
$ make

This compiles two executables:

ExecutablePurposeSource files
executables/omegaParticle-hole state density computationmain_omega.cpp, state_density.cpp, preeq_common.cpp
executables/lambdaInternal transition rate computationmain_lambda.cpp, transition_rates.cpp, preeq_common.cpp, state_density.cpp + MARLEY sources

Running Your First Calculation: omega

The omega executable computes particle-hole state densities for the n + Ar-40 system. Since it has no external dependencies beyond C++ math, it runs immediately:

$ ./executables/omega

This prints a YANDF-0.4 text table to stdout containing 25 rows of excitation energies (Ex = 1–25 MeV) with state densities for 9 exciton configurations, followed by a spin distribution table for n = 1–12.

To save the output:

$ ./executables/omega > omega_output.txt

Running Your First Calculation: lambda

The lambda executable computes internal transition rates and also requires the MARLEY environment variable to point to the MARLEY data directory:

$ MARLEY=../marley-private ./executables/lambda

By default this runs the KD03 validation mode (--mode kd03), which produces:

  1. A KD03 parameters log (lambda_printouts/OMP/KD03_Output/kd03_Ar40_n_*.log)
  2. A transition rates file (lambda_printouts/OMP/Rates_Output/lambda_Ar40_n_*.log)
Tip: Use make run to automatically set the MARLEY environment variable:
$ make run

Exploring the Output

Both executables produce YANDF-0.4 format (Yet Another Nuclear Data Format). The lambda output contains columns for four rate types across 21 exciton states:

ColumnMeaningUnits
p(p), h(p), p(n), h(n)Proton/neutron particle and hole numbers—
lambdapiplusProton pair creation rates-1
lambdanuplusNeutron pair creation rates-1
lambdapinuProton-to-neutron conversion rates-1
lambdanupiNeutron-to-proton conversion rates-1

Comparing with TALYS

The compare_lambda.py and compare_omega.py scripts compare two YANDF-0.4 output files and produce absolute difference and relative error reports:

$ python3 compare_lambda.py file1 file2 report.txt
$ python3 compare_omega.py file1 file2 report.txt
